Validate website URLs for reliability, security, and legitimacy. This endpoint performs comprehensive checks on web domains to assess their trustworthiness and identify potential risks.

## Endpoint

```
POST /kyc/web-validation
```

## Authentication

Requires `kyc:create` permission. Include your Bearer token in the Authorization header.

## Description

The web validation endpoint analyzes website URLs to determine their reliability and identify potential security or legitimacy concerns. It performs:

1. **Domain Analysis**: Validates domain structure and registration
2. **SSL/TLS Verification**: Checks certificate validity and security
3. **Content Analysis**: Evaluates website content and structure
4. **Risk Assessment**: Identifies potential security or fraud indicators
5. **Reliability Scoring**: Provides a reliability score (0-100)

## Request Body Parameters

<ParamFields>
  <ParamField
    name="url"
    type="string"
    required
    description="The website URL to validate (must include protocol: http:// or https://)"
  />
</ParamFields>

## Request Example

```bash
curl -X POST https://stg.kyc.legaltalent.ai/kyc/web-validation \
  -H "Authorization: Bearer YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{
    "url": "https://example.com"
  }'
```

## Response Format

### Success Response

```json
{
  "status": "success",
  "url": "https://example.com",
  "reliability_score": 85,
  "is_reliable": true,
  "checks": {
    "ssl_valid": true,
    "domain_valid": true,
    "content_analysis": "passed",
    "security_indicators": []
  },
  "warnings": [],
  "processing_time_ms": 3200,
  "timestamp": "2024-11-22T10:30:00Z"
}
```

### Response with Warnings

```json
{
  "status": "success",
  "url": "https://suspicious-site.com",
  "reliability_score": 45,
  "is_reliable": false,
  "checks": {
    "ssl_valid": true,
    "domain_valid": true,
    "content_analysis": "suspicious",
    "security_indicators": ["recent_registration", "low_reputation"]
  },
  "warnings": [
    "Domain registered less than 6 months ago",
    "Low reputation score detected"
  ],
  "processing_time_ms": 2800,
  "timestamp": "2024-11-22T10:30:00Z"
}
```

## Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `status` | string | Always `"success"` when validation completes |
| `url` | string | The validated URL |
| `reliability_score` | integer | Score from 0-100 (higher = more reliable) |
| `is_reliable` | boolean | `true` if reliability score >= 70 |
| `checks` | object | Results of individual validation checks |
| `warnings` | array | Array of warning messages (if any) |
| `processing_time_ms` | integer | Total processing time in milliseconds |
| `timestamp` | string | ISO 8601 timestamp of the validation |

### Checks Object

| Field | Type | Description |
|-------|------|-------------|
| `ssl_valid` | boolean | Whether SSL/TLS certificate is valid |
| `domain_valid` | boolean | Whether domain structure is valid |
| `content_analysis` | string | Content analysis result: `"passed"`, `"suspicious"`, or `"failed"` |
| `security_indicators` | array | Array of security indicator tags |

## Reliability Score Interpretation

| Score Range | Interpretation | Recommendation |
|-------------|----------------|----------------|
| 80-100 | Highly Reliable | Safe to proceed |
| 60-79 | Moderately Reliable | Review with caution |
| 40-59 | Low Reliability | Exercise caution |
| 0-39 | Very Low Reliability | Avoid or investigate further |

## Usage Examples

### Python Example

```python
import requests

token = "YOUR_TOKEN"

response = requests.post(
    "https://stg.kyc.legaltalent.ai/kyc/web-validation",
    headers={
        "Authorization": f"Bearer {token}",
        "Content-Type": "application/json"
    },
    json={
        "url": "https://example.com"
    }
)

if response.status_code == 200:
    data = response.json()
    print(f"Reliability Score: {data['reliability_score']}")
    print(f"Is Reliable: {data['is_reliable']}")
    
    if data['warnings']:
        print(f"Warnings: {', '.join(data['warnings'])}")
else:
    print(f"Error: {response.json()}")
```

### JavaScript Example

```javascript
const token = "YOUR_TOKEN";

async function validateWebsite(url) {
  const response = await fetch(
    "https://stg.kyc.legaltalent.ai/kyc/web-validation",
    {
      method: "POST",
      headers: {
        "Authorization": `Bearer ${token}`,
        "Content-Type": "application/json"
      },
      body: JSON.stringify({ url: url })
    }
  );
  
  const data = await response.json();
  
  if (response.ok) {
    console.log(`Reliability Score: ${data.reliability_score}`);
    console.log(`Is Reliable: ${data.is_reliable}`);
    
    if (data.warnings.length > 0) {
      console.log(`Warnings: ${data.warnings.join(', ')}`);
    }
    
    return data;
  } else {
    console.error("Error:", data.error);
    throw new Error(data.error);
  }
}
```

## Best Practices

- **Always include protocol**: URLs must include `http://` or `https://`
- **Validate before processing**: Check reliability score before making business decisions
- **Review warnings**: Pay attention to warning messages for additional context
- **Combine with other checks**: Use web validation alongside other KYC checks for comprehensive risk assessment
- **Handle timeouts**: Implement retry logic for production use (endpoint has 60-second timeout)

## Performance

- **Typical Response Time**: 2-5 seconds
- **Timeout**: 60 seconds maximum
- **Rate Limits**: Subject to API rate limiting (1,000 requests per 5 minutes)

## Integration Tips

1. **Pre-validation**: Validate URLs before allowing users to submit forms
2. **Batch Processing**: For multiple URLs, make separate requests (not supported in batch)
3. **Caching**: Consider caching results for frequently checked URLs
4. **Error Handling**: Implement proper error handling for network failures
